## テンプレートの使い方

### 1. プロジェクト情報のカスタマイズ

以下のファイルを編集して、プロジェクト固有の情報に置き換えてください：

- [astro.config.mjs](docs/astro.config.mjs) - サイトタイトル、ソーシャルリンク
- [index.mdx](docs/src/content/docs/index.mdx) - トップページのコンテンツ
- [CLAUDE.md](CLAUDE.md) - AIアシスタントへの指示（オプション）

### 2. 各セクションの編集

各ディレクトリには、テンプレートとなるサンプルファイルが含まれています。
プロジェクトに合わせて内容を書き換えてください。

#### Product セクション

- `vision.md` - ビジョンとブランド戦略
- `users.md` - ターゲットユーザーとペルソナ
- `roadmap.md` - リリース計画

#### Specs セクション

- `features.md` - 機能一覧と詳細仕様
- `screens.md` - 画面遷移図
- `glossary.md` - 用語集

#### System セクション

- `frontend.md` - フロントエンド設計
- `backend.md` - バックエンド設計
- `api.md` - API仕様
- `auth.md` - 認証・認可
- `infra.md` - インフラ構成
- `security.md` - セキュリティ要件

#### Development セクション

- `branch-strategy.md` - ブランチ戦略
- `progress.md` - 開発進捗管理
- `testing.md` - テスト戦略

### 3. 不要なファイルの削除

プロジェクトに不要なファイルやセクションは削除してください。

## 開発

### ローカルでの起動

```bash
cd docs
pnpm install
pnpm dev
```

ブラウザで `http://localhost:4321` にアクセス

### Docker での起動

```bash
cd docs
docker compose up
```

環境変数でプロジェクト名とポートをカスタマイズできます。詳細は [README.md](docs/README.md) を参照してください。

---

## カスタマイズのヒント

### サイドバーの調整

[astro.config.mjs](docs/astro.config.mjs) の `sidebar` 設定を編集：

```javascript
sidebar: [
  {
    label: "Product",
    autogenerate: { directory: "product" },
  },
  // 必要に応じて追加・削除・並び替え
],
```

### ドキュメント構造の変更

- 新しいディレクトリを追加したら、`astro.config.mjs` のサイドバーに追加
- `autogenerate` を使用すると、ディレクトリ内のファイルが自動でサイドバーに追加されます
