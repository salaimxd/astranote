> このセクションは、ユーザーの感情や思考を視覚的にマッピングし、貴重な顧客インサイトを得るための共感マップを管理します。

## 共感マップとは

共感マップは、**[ペルソナ](../persona/index.md)の感情や思考を6つの要素で視覚的にマッピング**し、ユーザーの潜在ニーズを発見するためのツールです。[ユーザーリサーチ](../user_research.md)で収集した実際のデータに基づいて作成し、表面的な要望の奥にある真のニーズを理解するために使用します。

## 作成の5ステップ

1. **ペルソナを定義する** - 実際の顧客データや調査情報を使用
2. **理想的な結果を定義する** - 目標や成果物を明確化
3. **共感マップを埋めていく** - 6つの要素すべてに情報を記入
4. **プロジェクトチームと共有する** - 全メンバーで同じペルソナイメージを共有
5. **共感マップを更新する** - 新しいデータに応じて継続的に更新

## 6つの構成要素

1. **考えていること・感じていること（Think & Feel）** - ペルソナの心理状態
2. **見ていること（See）** - 視覚的に接触するタッチポイント
3. **言っていること・していること（Say & Do）** - 日常的な言動と行動
4. **聞いていること（Hear）** - 周囲の人やメディアからの情報
5. **悩み・ストレスになっていること（Pain）** - ネガティブな悩みやストレス
6. **あったら嬉しいこと（Gain）** - 潜在的に望む状況やモノ

## 共感マップの作成方法

1. `_template.md` をコピーする
2. ファイル名を `persona_name.md`（例: `tanaka_taro.md`）に変更
3. [ペルソナ](../persona/index.md)と[ユーザーリサーチ](../user_research.md)の結果を参照しながら記入
4. 上記の「共感マップ一覧」テーブルに追加
5. `_meta.json` にファイル名を追加

## 横断的インサイト

4つの共感マップから得られた共通パターン：

### 共通Pain（ペインポイント）

- **翻訳の壁**: 作家（コスト）・読者（品質・速度）双方で翻訳が最大の障壁
- **プラットフォームの限界**: なろう/カクヨムは日本語圏限定、Webnovelは課金・品質に問題
- **収益構造の不満**: 作家への還元率が低く、読者も高額課金を強いられる
- **日本語読書体験の未成熟**: 検索機能不在、縦書き未対応、バッテリー消費、クラッシュなど基本品質の問題（高橋結衣の視点）

### 共通Gain（潜在ニーズ）

- **「書くだけで世界に届く」体験**: AI翻訳による言語障壁の解消
- **適正な対価**: 作家に直接届く収益モデル、読者に適正な価格設定
- **ワンストップ化**: 執筆→翻訳→公開→収益化をシームレスに
- **没入型読書体験**: 縦書き・明朝体・安定性など、日本語読書の基本品質の改善（高橋結衣の視点）
