## 資金計画とランウェイ

MVP開発から正式リリースまでの期間（推定6〜8ヶ月）は収益がほぼゼロの「ランウェイ期間」となる。事業継続のために資金計画を明確にする。

### 月間固定費の内訳（ランウェイ期間）

| 費目 | 月額概算 | 備考 |
|------|---------|------|
| 開発者人件費 | ¥0〜300,000 | ブートストラップ想定（副業・自己投資）。外部エンジニア採用時は最大¥300K |
| インフラ・サーバー | ¥10,000〜30,000 | MVP期は最小構成（Vercel/Railway等の低コストPaaS） |
| AI API費用（開発・テスト用） | ¥5,000〜20,000 | GPT-4o API、テスト翻訳・品質検証用 |
| ドメイン・SaaS | ¥5,000〜15,000 | ドメイン、メール、分析ツール（Amplitude等） |
| 法務費用（初回） | ¥50,000〜200,000 | 利用規約・プライバシーポリシー作成（初回のみ） |
| **月間固定費（ランウェイ期間合計）** | **¥70,000〜565,000/月** | 人件費の考え方次第で大きく変動 |

> **前提:** ブートストラップ（個人資金）での開発を想定。エンジニア1〜2名が副業・個人開発として参加するケースでは、人件費を大幅に抑制できる。

### ランウェイの計算

| シナリオ | 月間固定費 | 予算 | ランウェイ期間 |
|---------|-----------|------|--------------|
| **最小構成**（個人開発・副業） | ¥70,000〜100,000/月 | ¥500,000 | **5〜7ヶ月** |
| **標準構成**（パートタイム雇用1名追加） | ¥200,000〜300,000/月 | ¥500,000 | **1.7〜2.5ヶ月** |
| **フルタイム構成**（正社員2名） | ¥1,500,000〜1,680,000/月 | ¥500,000 | **約0.3ヶ月（要外部調達）** |

**MVP期間（6〜8ヶ月）を¥500,000で完走するには最小構成が前提条件。**

### 資金調達方針

| 調達方法 | 金額感 | 前提条件 | 優先度 |
|---------|--------|---------|--------|
| ブートストラップ（個人資金） | ¥300,000〜500,000 | 副業・節約で確保 | **最優先**（MVP期） |
| 個人資金の追加投入 | ¥500,000〜1,000,000 | 収益実績・MVP検証結果に応じて | 第2優先 |
| 補助金・助成金（IPA等） | ¥500,000〜5,000,000 | 申請要件の確認が必要 | 第3優先 |
| エンジェル投資・VC | ¥5,000,000〜50,000,000 | MVP検証済み・初期トラクション必要 | 第4優先（β版以降） |

### 資金枯渇リスクへの対策

| リスク | 発生条件 | 対策 |
|--------|---------|------|
| MVP開発が想定より長引く | 技術的難易度の過小見積もり | スコープをさらに削減（翻訳のみのα版からスタート） |
| 初期ユーザー獲得が遅い | チキンエッグ問題が解決できない | 作家へのコンシェルジュ翻訳サービスで有料化し、キャッシュを確保 |
| APIコストが予想を超える | 翻訳量・モデル選定の誤り | Haiku等の低コストモデルに一時的に切り替え、品質を維持できるか検証 |
| 資金が尽きる | ランウェイ計算の誤り | MRRが¥100,000を超えた時点で外部調達を開始。それ以前は支出を最小化 |

**ランウェイ管理の目安:** 残ランウェイが3ヶ月を切った時点で、外部調達または事業縮小の判断を行う。

## 市場規模の算出根拠

本セクションでは、リーンキャンバスの競合優位性欄に記載した市場規模（TAM $15.2B / SAM $2.1B / SOM $21M）の算出根拠を明記する。

### TAM（Total Addressable Market）: $15.2B

**グローバルデジタルコンテンツ市場（Web小説・ライトノベルジャンル）**

| 前提条件 | 数値 | 出典・根拠 |
|----------|------|-----------|
| グローバルのWeb小説市場規模（2024年） | 約$12〜16B | 業界調査レポート（Wattpad/Webnovelの公開情報 + アナリスト推定） |
| ラノベ・アニメ関連コンテンツの成長率（年率） | 約15〜20% | アニメ産業レポート（日本動画協会等）より推定 |
| 採用数値 | $15.2B（中央値） | 複数ソースの中央値を採用 |

**前提条件:**

- 「グローバルデジタルWeb小説市場」全体を対象とする（日本語・英語・中国語等を含む）
- 電子書籍全体（$18B+）ではなく、連載型Webコンテンツ（Webnovel、Wattpad等が競合するセグメント）に限定
- 紙の書籍市場は含まない

### SAM（Serviceable Addressable Market）: $2.1B

**英語圏のラノベ・アニメファン向けデジタルコンテンツ市場**

| 前提条件 | 数値 | 根拠 |
|----------|------|------|
| 英語圏（米・英・豪・加）のアニメ・ラノベファン推定人口 | 約5,000万人 | Crunchyroll等の公開ユーザー数・業界データを参照 |
| うち有料デジタルコンテンツに課金する割合 | 約30% | アプリストアレビュー分析・競合の有料プラン普及率から推定 |
| ARPU（年間平均課金額） | 約$14/年（$1.2/月） | Webnovelの課金体系（月$50+）の低コスト代替として$5/月の保守的な普及率を想定 |
| SAM合計 | 5,000万 × 30% × $14 ≈ $2.1B | |

**前提条件:**

- 英語を主要言語とする市場に限定（astranoteのMVPは日→英翻訳のみ）
- Webnovel・Wattpad等の既存プラットフォームが既にサービスを提供している市場セグメント
- 将来的な多言語展開（中国語・韓国語・スペイン語）はSAMに含まない

### SOM（Serviceable Obtainable Market）: $21M

**astranoteが現実的に獲得できる市場シェア（3〜5年後）**

> **注記（2026-02-17修正）:** 旧版では投げ銭GMV $30Mを別途計上していたが、これは読者サブスク収益と二重計上になるため削除し、正しい積み上げ計算に修正した。また、SAMのARPU（$14/年）はWebnovelの月$50+に対する「低コスト代替として$1.2/月が普及した場合」の保守的な前提であり、astranoteの読者プランの$5/月とは別の想定値であることを明記した。

| 前提条件 | 数値 | 根拠 |
|----------|------|------|
| SAM（$2.1B）に対するシェア率 | 1% | スタートアップの市場参入1〜3年目の現実的シェア |
| 作家サブスク収益 | ¥1,000/月 × 12ヶ月 × 作家1,000名 = ¥12,000,000（≈$80,000） | 損益分岐点（400名）超の成長想定 |
| 読者サブスク収益 | $5/月 × 12ヶ月 × 読者15,000名 = $900,000 | リーンキャンバスの12ヶ月目標値（15,000名）から算出 |
| 投げ銭手数料収益（修正後） | 読者15,000名 × 投げ銭参加率8% × ¥1,000/月 × 12ヶ月 × 30%手数料 = 約¥4,320,000（≈$29,000） | 読者15,000名を母数とした正しい計算（旧版のGMV $30Mは二重計上のため修正） |
| SOM年間収益合計（12ヶ月目標ベース） | 約$980,000（≈$1M） | 上記3つの積み上げ |
| SAM 1%シェア換算（$2.1B × 1%） | $21M | 3〜5年後の中長期目標。12ヶ月目標の約21倍規模。多言語展開・B2B連携含む成長ロードマップで到達想定 |
| SOM採用値 | $21M | 3〜5年後の中長期目標として維持（SAM 1%シェア） |

**SOMとARPUの補足:**

- SAMの算出に用いたARPU（$14/年≈$1.2/月）は「SAM全体の平均的な課金額」であり、Webnovel月$50+の利用者を含む市場全体の保守的な平均値
- astranoteの読者プラン（$5/月=$60/年）はSAM平均ARPUより高く、有料ユーザー比率が高まるほどSOMは上方修正される
- 12ヶ月後の年間収益（約$1M）と3〜5年後のSOM目標（$21M）のギャップは、多言語展開・追加チャネル・B2B連携による規模拡大で埋める計画

**前提条件:**

- SAM（$2.1B）の1%シェアが3〜5年以内に現実的な到達目標
- なろう系・ラノベ系特化で深いニッチを確保する戦略
- 1%シェア達成後の拡大フェーズ（多言語展開・B2B出版社連携）はSOMに含まない
- 為替レートは$1 = ¥150で計算

**注意事項:** これらの数値は推定値であり、独立した市場調査機関による検証を推奨する。MVP検証段階では、SOMの前提条件（作家1,000名・読者15,000名）の達成可能性をLPのCVR・β版の成長率から継続的に検証する。

## チキンエッグ問題の解決戦略

astranoteはプラットフォームビジネスであり、作家（コンテンツ供給側）と読者（コンテンツ需要側）の両方を同時に獲得しなければならないチキンエッグ問題に直面する。このセクションでは、どちらを先に獲得するかの明確な優先順位と、具体的な初期獲得戦術を定義する。

### 優先順位: 作家ファーストで獲得する

**作家を先に獲得する理由:**

| 理由 | 説明 |
|------|------|
| コンテンツがなければ読者は来ない | 読者が訪問した際にコンテンツが存在しないと、即座に離脱する。コンテンツを先に準備することが必須 |
| 作家のモチベーションが持続する | 自分の作品が英語に翻訳された体験は強力なWoW体験。作家が積極的にSNSでシェアし、読者獲得を促進する |
| 作家のLTV > 読者のLTV | 作家のLTV（¥24,000）は読者（$90）より高く、先行投資のROIが高い |
| 読者獲得はコンテンツ量に依存 | 読者は「読める作品が多い」ことで継続する。作家30名→作品30作品以上が揃った段階で本格的な読者獲得を開始する |

### フェーズ別の獲得戦術

**フェーズ1: 作家獲得（MVP開発期間中〜α版）**

| 戦術 | 具体的アクション | KPI |
|------|----------------|-----|
| ターゲット作家への直接アプローチ | なろう週間ランキング上位300〜500作品の作家に個別DM。「英語翻訳を5分でできます」の価値提案 | 返信率1〜3%（業界標準）。300名にDMで3〜9名、500名で5〜15名の返信を期待。目標5名確保のためDM母数を上方修正 |
| ファンサブコミュニティとの連携 | r/noveltranslations、Translation Communities Discord への投稿。「AI翻訳ツールのβテスターを募集」 | β版参加登録20名以上 |
| α版コンシェルジュ翻訳 | 最初の5作品については、プロンプトを手動調整した「コンシェルジュ翻訳」で品質を保証。口コミの起点を作る | 翻訳品質スコア4.0以上 |

**フェーズ2: 読者獲得（β版以降、作品30本以上が揃った段階）**

| 戦術 | 具体的アクション | KPI |
|------|----------------|-----|
| 作家から読者へのクロスプロモーション | 翻訳公開した作家がX（Twitter）でシェア。作家のフォロワーが読者の第一波となる | 作家1人あたり読者5名以上の流入 |
| Reddit・Discordでの存在感構築 | r/LightNovels で翻訳サンプルを公開し、コミュニティの反応を測定 | スレッド閲覧数1,000以上 |
| 読者→作家の紹介ループ | 英語読者が「翻訳してほしい作品」をリクエストする機能。日本語作家への需要シグナルを可視化 | リクエスト総数50件以上 |

### 自己強化ループの設計

作家と読者が互いを引き寄せる自己強化ループを以下の構造で設計する。

```
作家がastranoteで執筆・翻訳
        ↓
英語翻訳作品が公開される
        ↓
海外読者がSNS/Reddit/Discordで発見・シェア
        ↓
読者増加 → 読者から作家への「応援投げ銭」
        ↓
作家の収益増加 → 「astranoteで書けば海外読者が増える」という口コミ
        ↓
新規作家がastranoteに登録
        ↓
コンテンツが増え、読者がさらに増加（ループ）
```

**書き手が同時に読み手になる仕組み:**

多くのWeb小説作家は同時に熱心なラノベ読者でもある。astranoteでは「自分が翻訳した作品」と「他の作家が翻訳した作品」を同じプラットフォームで読める体験を提供することで、作家がそのまま読者コミュニティの核となる。

- 作家の「読者ダッシュボード」で他作品を読める機能（β版以降）
- 作家同士のフォロー機能で「相互読者」コミュニティを形成
- 作家が翻訳した他作家の作品を「おすすめ」できる機能

## 法的リスク分析

> **重要:** このセクションの内容は、MVP開発開始前に法務専門家によるレビューを受けることを強く推奨する。以下はリスクの初期整理であり、法的助言ではない。

### リスク1: なろう・カクヨムの利用規約（P0）

**リスク内容:**

小説家になろう・カクヨムの利用規約は、ユーザーが投稿した作品の著作権はユーザー（作家）自身に帰属するとしている。しかし、外部サービスへのテキスト転送・API経由での利用については規約上の解釈が曖昧な部分がある。

| リスク項目 | 内容 | 現時点の評価 |
|-----------|------|------------|
| テキストの外部転送 | 作家が自身の作品テキストをastranoteにコピー&ペーストするフローは、著作権者（作家自身）が行う操作であるため合法と解釈できる | 低リスク（作家自身の操作） |
| スクレイピング・API経由の自動取得 | なろう/カクヨムからの自動取得は利用規約違反の可能性が高い | **高リスク（禁止事項）** |
| 「転載禁止」作品の扱い | 作家が「転載禁止」に設定した作品でも、作家自身がコピーする分には問題ない | 低リスク |

**対策:**

- **設計上の対策**: astranoteでは、作家が自身でテキストをコピー&ペーストするフローのみを提供する。スクレイピング・自動同期機能はMVPスコープから除外する
- **利用規約の明確化**: astranoteの利用規約で「作家自身が権利を持つコンテンツのみアップロード可能」と明記する
- **法務レビュー計画**: MVP開発開始前に、なろう/カクヨムの最新利用規約を法務専門家がレビューする（2026年3月までに実施）

### リスク2: 書籍化作家の出版社との独占契約（P1）

**リスク内容:**

書籍化した作家の中には、出版社との間で「海外出版権の独占ライセンス」を付与する契約を締結しているケースがある。この場合、作家自身が第三者（astranote）を通じて英語翻訳・公開を行うことが契約違反となる可能性がある。

| リスク項目 | 影響 | 発生確率 |
|-----------|------|---------|
| 出版社との海外出版権独占契約 | 対象作品の公開停止・法的紛争 | 中（書籍化作家をターゲットにする場合） |
| アニメ化作品の権利関係 | メディアミックス作品は権利が複雑化する傾向がある | 高（人気作品ほどリスク大） |
| 出版社との事後請求 | 作品の人気が出た後に出版社が権利主張する可能性 | 低〜中 |

**対策:**

- **MVP期間の方針**: アーリーアダプターとして「書籍化経験のない作家、またはデジタル権利を自身で保有することを確認した作家」を優先的に選定する
- **オンボーディング時の確認**: 作家の登録時に「本作品の海外出版権について、第三者への許諾に制限がないことを確認してください」のチェックボックスを設ける
- **法務レビュー計画**: 書籍化作家へのアプローチを開始する前に（正式リリース前）、出版社との契約形態について法務専門家に確認する

### リスク3: AI翻訳物の著作権帰属問題（P1）

**リスク内容:**

AI翻訳によって生成されたテキストの著作権帰属については、各国で法的解釈が分かれており、2026年2月時点でも議論が続いている。

| 国・地域 | 現時点の状況 | astranoteへの影響 |
|---------|-------------|-----------------|
| 日本 | AI生成物の著作権についての明確な法令なし。人間の創作的関与がある場合は著作権が認められる傾向 | 作家がプロンプト設定・辞書入力等の創作的関与を行うことで、著作権は作家に帰属すると解釈できる可能性が高い |
| 米国 | 著作権局はAI単体が生成した著作物を著作権保護の対象外と明示。人間の創作的関与が必要 | 翻訳物の著作権主体が曖昧になるリスク。「AI翻訳の結果を作家が監修・承認した」という記録を保持する |
| EU | AI法（EU AI Act）の施行により、高リスクAIシステムへの規制が強化されつつあるが、著作権については未整備 | 将来的なEU展開時に著作権帰属の明確化が必要 |

**対策:**

- **短期的対策（MVP期間）**: 翻訳物の著作権は「原作者（作家）」に帰属することをastranoteの利用規約で明記する。astranoteはあくまで「翻訳ツールを提供するプラットフォーム」であり、翻訳物の著作権を主張しない
- **作家への情報提供**: オンボーディング時に「AI翻訳物の著作権については各国で議論中であり、公開前に作家自身で確認することを推奨する」旨を案内する
- **翻訳履歴の保持**: いつ誰が何をプロンプトとして設定し、どの翻訳結果を承認したかのログを保持する（将来的な著作権紛争に備えた証拠）

## 仮説の検証計画

### 最もリスクの高い仮説トップ3

#### 仮説1: AI翻訳の品質がラノベ読者に受け入れられる（価値仮説・最重要）

| 項目 | 内容 |
|------|------|
| 仮説 | ラノベ特化AI翻訳は、ファンサブ翻訳と同等以上の品質を実現し、読者の70%以上が「十分に読める」と評価する |
| リスクレベル | **最高** --- プロダクトの根幹。翻訳品質が低ければ全てが崩壊する |
| 検証方法（MVP接続） | MVP v1でAI翻訳を実装し、既存のファンサブ翻訳と品質比較テスト（A/Bテスト）を実施。30名の読者に5作品の翻訳を評価してもらう |
| 成功基準 | 読者の70%以上が「ファンサブ翻訳と同等以上」と評価、翻訳品質スコア4.0/5.0以上 |
| 検証期間 | 4週間 |
| 失敗時のピボット | 人間翻訳者とのハイブリッドモデル（AI翻訳+人間レビュー）に切り替え |

#### 仮説2: 作家がAI翻訳に月¥1,000を支払う意思がある（成長仮説）

| 項目 | 内容 |
|------|------|
| 仮説 | なろう・カクヨムで連載中の作家の20%以上が、海外展開のためにAI翻訳サービスに月¥1,000を支払う意思がある |
| リスクレベル | **高** --- 収益モデルの基盤。無料でなければ使わない可能性 |
| 検証方法（MVP接続） | ランディングページで事前登録を募集し、¥1,000/月の価格を明示。50名の作家にベータテスト招待を送り、有料転換率を計測 |
| 成功基準 | ランディングページCVR 5%以上、ベータテスターの有料転換率20%以上 |
| 検証期間 | 6週間 |
| 失敗時のピボット | フリーミアムモデル（基本無料・プレミアム翻訳を有料）に切り替え |

#### 仮説3: 読者が月$5でサブスクリプションに加入する（成長仮説）

| 項目 | 内容 |
|------|------|
| 仮説 | Webnovelに月$50以上を支払っている読者の40%以上が、月$5の読み放題プランに乗り換える |
| リスクレベル | **中〜高** --- 読者側の収益基盤。コンテンツ量が不十分だと加入しない |
| 検証方法（MVP接続） | MVP公開後、30作品以上を揃えた段階で読者向けサブスク開始。無料トライアル（14日間）から有料転換率を計測 |
| 成功基準 | 無料トライアル→有料転換率15%以上、月間チャーン率10%以下 |
| 検証期間 | 8週間 |
| 失敗時のピボット | 作品単位の個別課金モデル（1話¥50〜100）に切り替え |

## Discoveryとの整合性チェック

| リーンキャンバス要素 | 参照元ドキュメント | 整合性 |
|---|---|---|
| 顧客セグメント（作家2名+読者1名） | [ペルソナ](../../01_discovery/persona/index.md)（山田太郎、佐藤美咲、Emily Chen） | 3名のペルソナを作家・読者の2セグメントに統合。プライマリ（作家）とセカンダリ（読者）の優先度を維持 |
| 課題（翻訳の壁・収益構造・プラットフォーム老朽化） | [共感マップ Pain](../../01_discovery/empathy_map/index.md) | 共感マップの7つの作家Pain・4つの読者Painを3つの戦略的課題に集約。アプリストアレビュー600件+で検証済み |
| 独自の価値提案（Write Once, Read Anywhere） | [HMW](../../01_discovery/hmw.md)の3つの戦略的質問 | 3つのHMW（翻訳の壁・エコシステム・創作体験）を1つのUVPに統合 |
| 解決策（AI翻訳・収益モデル・モダンエディタ） | [HMW](../../01_discovery/hmw.md)の解決策方向性 | HMWの解決策方向性をそのまま3つの主要機能に変換。MVPスコープを設定 |
| チャネル | [ユーザーリサーチ](../../01_discovery/user_research.md) | ユーザーリサーチで特定したターゲットの行動場所（なろう、Reddit、Discord）をチャネルに反映 |
| 収益の流れ | [競合分析](../../01_discovery/competitive_analysis.md) | 競合の課金体系（Webnovel月$50+）に対する差別化価格として設定。70%作家還元は業界最高水準 |
| コスト構造 | --- | 新規作成。AI API利用料の変動費モデルを採用し、スケーラビリティを確保 |
| 主要指標 | [ユーザーリサーチ](../../01_discovery/user_research.md) | ユーザーリサーチで検証済みの課題解決度を測るAARRR指標を設定 |
| 競合優位性 | [競合分析](../../01_discovery/competitive_analysis.md)（Wattpad、Webnovel、なろう、カクヨム） | 4社の競合分析結果を「グローバル×ラノベ特化×作家ファースト」の未占領ポジションに集約。TAM/SAM/SOMを引用 |
