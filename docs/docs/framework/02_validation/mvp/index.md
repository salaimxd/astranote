## チェックリスト

### 計画フェーズ

- [x] 仮説を明確に定義した（セクション2: 価値仮説・成長仮説・最重要仮説）
- [x] ターゲットユーザーを特定した（セクション3: 山田太郎タイプ、アーリーアダプター30名+読者100名）
- [x] MVPタイプを選択した（セクション4: シングル機能型MVP）
- [x] 成功基準を設定した（セクション6: 翻訳品質4.0/5.0以上、完了率40%以上、完読率30%以上）
- [x] リスクを洗い出した（セクション7: 翻訳品質不足、ラインナップ不足、スケーラビリティ等5件）

### 開発フェーズ

- [ ] 翻訳品質POCを完了した（5作品×5話、英語読者10名、Go/No-Go判定済み）
- [ ] 最小限の機能に絞った（P0機能6つ: AI翻訳、エディタ、公開機能、認証、翻訳品質評価UI、オンボーディングフロー）
- [ ] 技術スタックを決定した（Astro + Cloudflare Workers/D1/KV/R2 + Gemini Flash/Claude Haiku）
- [ ] 開発マイルストーンを設定した（POC + M1〜M4、10〜12週間）
- [ ] 品質基準を定義した（レスポンス10秒以内、稼働率99.5%、WCAG 2.1 AA）
- [ ] 法的前提条件チェックリストを確認した（なろう/カクヨム利用規約、著作権帰属、個人情報保護）

### リリースフェーズ

- [ ] アーリーアダプターをリクルートした（作家30名、読者100名）
- [ ] 計測ツールを設置した（Mixpanel、Hotjar、Googleフォーム）
- [ ] サポート体制を整えた（Discord or GitHub Discussions）
- [ ] リリース前チェックを完了した（開発/デザイン/マーケティング/運用）

### 検証フェーズ

- [ ] 定量データを収集した（翻訳品質スコア、ファネル、リテンション）
- [ ] 行動を観察した（Hotjarセッション録画、翻訳修正ログ）
- [ ] フィードバックを集約した（アンケート、インタビュー、NPS）

### 評価フェーズ

- [ ] データを分析した（目標値 vs 実績値の比較）
- [ ] 学びを文書化した（セクション11に記録）
- [ ] 仮説の検証結果をまとめた（正しかった/部分的に正しかった/間違っていた）
- [ ] 次のアクションを決定した（継続/ピボット/中止）
