## 1. 背景と目的

### 1.1 背景

**プロジェクトの背景:**

Discoveryフェーズにおいて、以下の6つのドキュメントを完成させ、Web小説のグローバル展開における課題を深く理解した。

1. **ユーザーリサーチ** — 作家・読者双方の生の声を収集し、翻訳コスト・品質・プラットフォームの課題を特定
2. **競合分析** — Syosetu/カクヨム/Webnovel/DeepL等の4社を分析し、「ラノベ特化AI翻訳×作家還元」の空白市場を発見
3. **ペルソナ** — 山田太郎（作家・プライマリ）、Emily Chen（海外読者・セカンダリ）、佐藤美咲（プロ作家・セカンダリ）の3名を定義
4. **共感マップ** — 各ペルソナの思考・感情・行動パターンを可視化し、痛みと期待を明確化
5. **ジャーニーマップ** — 「執筆→翻訳→公開→読者獲得→収益化」の全プロセスにおけるペインポイントをマッピング
6. **HMW（How Might We）** — 「どうすれば翻訳の壁を壊せるか」「どうすれば作家に正当な収益を届けられるか」「どうすれば読者の没入体験を実現できるか」の3つの問いを設定

**解決したい課題:**

| 課題 | 内容 | 影響 |
|------|------|------|
| 翻訳の壁 | 日本語Web小説の英語翻訳コストが1作品50〜100万円、期間6ヶ月以上 | 99%以上の作品が海外読者に届かない |
| 収益構造の歪み | 既存プラットフォームの印税率5〜15%、投げ銭手数料50%以上 | 作家の創作モチベーション低下 |
| プラットフォーム老朽化 | なろう/カクヨムのUI/UXが2010年代のまま、海外読者向け機能なし | グローバル展開の障壁 |

### 1.2 MVPの目的

**このMVPで検証したいこと:**

- [x] ラノベ特化AI翻訳が、ファンサブ翻訳と同等以上の品質（4.0/5.0以上）を実現できるか
- [x] なろう/カクヨムの作家が月¥1,000の翻訳サービスに支払意思を持つか
- [x] 海外ラノベ読者がAI翻訳作品を「十分に読める」と感じ、継続的に読むか

**期待される学び:**

- AI翻訳のどの側面（語彙の正確さ、文体の再現性、キャラクターの一人称、固有名詞の一貫性）が読者の没入感に最も影響するかを理解する
- 作家が翻訳サービスに求める最低限の機能セットを特定する
- 海外読者の作品発見から継続読書に至るユーザージャーニーを実データで検証する

## 翻訳品質POC（事前技術スパイク）

> **技術レポート最優先提言:** MVP本開発に入る前に、2週間の集中的な翻訳品質POC（技術スパイク）を実施する。「AI翻訳品質がラノベ読者に受け入れられる」という最重要仮説を、本開発コストをかける前に検証する。POCで目標スコア（4.0/5.0）を達成した場合のみ、MVP本開発に移行する。

### POCの目的

本開発（10〜12週間）に入る前に、以下の技術的実現可能性を検証する。

1. **ラノベ特有の文体に特化したプロンプトでGPT-4/Claude APIが4.0/5.0以上を実現できるか**
2. **どのプロンプト戦略が最も高スコアを出すか**（比較評価）
3. **翻訳APIコストの実測値を取得する**（コスト構造の見直しに活用）

### POCの実施計画

**期間:** 2週間（MVP本開発前、2026-02-17 〜 2026-02-28）

**評価対象:**

| 評価軸 | 内容 |
|--------|------|
| 評価作品数 | 5作品（ジャンル分散: 異世界転生・ファンタジー・日常系・ラブコメ・スポーツ） |
| 評価話数 | 各作品5話（合計25話） |
| プロンプト戦略数 | 3〜5パターン（ベースライン・キャラクター辞書あり・文体指定あり・コンテキスト保持あり・組み合わせ）を比較 |
| 比較対象 | 同一作品の既存ファンサブ翻訳（ブラインドテスト用） |

**評価方法:**

| 評価方法 | 内容 | 人数 |
|---------|------|------|
| 英語読者によるブラインドテスト | AI翻訳とファンサブ翻訳を区別せずに提示し、5段階評価+コメントを収集 | 10名（最低限）推奨20名 |
| 翻訳専門家によるレビュー | ラノベ翻訳経験のある翻訳者2名による品質評価（語彙・文体・固有名詞・一貫性） | 2名 |
| 定量スコア集計 | 各プロンプト戦略の平均スコアを算出し、最優秀プロンプトを特定 | - |

> **サンプルサイズの注記（技術レポート P2 対応）:** n=10の評価では信頼区間が±14%程度と広く、スコア4.0は統計的に3.86〜4.14の範囲を意味する。Go/No-Go判定の信頼性を高めるために、可能であればn=20への拡大を推奨する。n=20の場合、信頼区間は±10%程度に縮小し、意思決定の確実性が向上する。テスター謝礼予算に余裕がある場合は優先的にn=20で実施する。

**成功基準（Go/No-Go判定）:**

| 判定 | 条件 | 次のアクション |
|------|------|--------------|
| Go（本開発へ移行） | 最優秀プロンプトの翻訳品質スコアが4.0/5.0以上 | 2026-03-01からMVP本開発を開始 |
| 要改善（条件付きGo） | スコア3.5〜3.9。追加プロンプト改善で4.0到達の見込みあり | 1週間の追加チューニング後に再評価 |
| No-Go（ピボット検討） | スコア3.5未満。プロンプト改善だけでは限界の可能性 | 人間翻訳者+AIのハイブリッドモデルへのピボットを検討 |

**POC期間中のコスト目安:**

| 項目 | 概算 | 備考 |
|------|------|------|
| AI翻訳API費用 | ¥10,000〜20,000 | 25話 × 5,000文字 × 5プロンプト × ¥0.5〜1.5/1,000文字 |
| テスター謝礼 | ¥15,000〜20,000 | 英語読者10名 × ¥1,500〜2,000 |
| 翻訳専門家費用 | ¥10,000〜30,000 | 翻訳者2名のレビュー報酬 |
| POC合計 | 約¥35,000〜70,000 | 本開発予算¥500,000の7〜14% |

### POC結果による計画への影響

POCの結果は、MVPの機能スコープ・技術スタック・予算配分に直接影響する。

| POC結果 | 影響 |
|---------|------|
| Go（スコア4.0以上） | 計画通りにMVP本開発を開始 |
| 要改善（3.5〜3.9） | M2（翻訳エンジン）の工数を+1週間追加、予算をAPIチューニングに+¥20,000配分 |
| No-Go（3.5未満） | ソリューションをピボット。コンシェルジュ型MVP（人間翻訳+AI補助）に切り替えを検討 |

## 2. 仮説定義

### 2.1 価値仮説（Value Hypothesis）

**誰が:** 山田太郎タイプ（なろう/カクヨムで連載中、累計10万PV以上、海外展開に興味があるがコスト障壁で諦めている作家）

**何に困っていて:** 翻訳コスト50〜100万円・期間6ヶ月以上の障壁により、自分の作品を海外読者に届けることができない。既存の機械翻訳（Google翻訳・DeepL）ではラノベ特有の文体（一人称の使い分け、異世界固有名詞、ルビ表現）が正しく翻訳されず実用に耐えない。

**何を提供すれば:** ラノベ特化AI翻訳エンジン（月¥1,000で無制限翻訳、ワンクリックで日→英翻訳、キャラクター名辞書・文体設定対応）

**どうなるか:** 作家が月¥1,000で自分の作品を即座に英語圏の読者に届けられるようになり、「自分の作品が世界で読まれている」という体験を得る。Emily Chenタイプの海外読者が「borderline unreadable」なファンサブ翻訳から「没入できる品質」の翻訳を得られるようになる。

**検証方法:**

- A/Bテスト: AI翻訳 vs 既存ファンサブ翻訳の品質比較（読者によるブラインド評価）
- 翻訳品質スコア: 5段階評価アンケート（目標4.0/5.0以上）
- 作家の初回翻訳完了率: 登録→翻訳実行までの完了率（目標40%以上）

### 2.2 成長仮説（Growth Hypothesis）

**どのチャネルで:** X（Twitter）の日本語Web小説作家コミュニティ、Reddit r/LightNovels・r/noveltranslations、Discord翻訳コミュニティ

**どのように顧客を獲得し:**

- 作家側: X（Twitter）でのβテスター募集、なろう/カクヨムの上位ランキング作家への直接DM、作家コミュニティでの口コミ
- 読者側: Reddit/Discordでの翻訳作品公開、英語圏ラノベレビューサイトとの連携

**どのように収益化するか:**

- 作家: 月額¥1,000のサブスクリプション（翻訳枚数無制限）
- 読者: 月額$5のサブスクリプション（全作品読み放題）
- 投げ銭: 30%プラットフォーム手数料、70%作家還元

**検証方法:**

- LP登録率: β版LP訪問→メール登録の転換率（目標5%以上）
- 作家アンケート: 月¥1,000の支払意思（目標60%以上が「払う」と回答）
- 読者無料→有料転換率: 無料期間後のサブスク加入率（目標5%以上）

### 2.3 最もリスクの高い仮説

**このMVPで最優先で検証すべき仮説:**

> ラノベ特化AI翻訳が、ファンサブ翻訳と同等以上の品質を実現し、海外読者の70%以上が「十分に読める」と評価する

**なぜこれが最もリスクが高いか:**

この仮説はastranoteのプロダクト価値の根幹を成す。AI翻訳の品質が不十分であれば、作家は翻訳結果を公開したがらず（供給サイドの崩壊）、読者は離脱する（需要サイドの崩壊）。価格や機能の問題はピボット可能だが、翻訳品質の問題はプロダクトの存在意義そのものに関わる。リーンキャンバスで特定した3つの仮説（AI翻訳品質、作家の支払意思、読者のサブスク加入）のうち、他の2つは価格調整やモデル変更で対応可能だが、翻訳品質の問題は技術的な根本課題であり、後からの修正コストが最も高い。

## 3. ターゲットユーザー

### 3.1 アーリーアダプター

**具体的なペルソナ:** 山田太郎タイプ — なろう/カクヨムで連載中のWeb小説作家

**属性:**

- 年齢・性別: 25〜35歳、男性が中心（Web小説作家の主要層）
- 職業: ITエンジニア、クリエイター等（本業+副業で執筆）
- 居住地: 日本国内（首都圏中心）
- デジタルリテラシー: 高い（日常的にWebサービスを使いこなす、新しいツールへの抵抗感が低い）

**行動特性:**

- なろう/カクヨムで累計10万PV以上、定期連載中（週1〜3回更新）
- X（Twitter）で作家活動を発信、フォロワー1K〜5K
- 海外からのコメントやファンレターを受け取った経験があり、海外展開に関心がある
- 翻訳コストの高さから海外展開を諦めていた（50〜100万円は副業収入では捻出困難）
- 新しいWebサービスやAIツールに積極的に試す「イノベーター」気質

**獲得方法:**

- X（Twitter）でのβテスター募集ツイート（ハッシュタグ #小説家になろう #カクヨム #Web小説 を活用）
- なろう/カクヨムの週間ランキング上位作家へのDM（直接アプローチ）
- Web小説関連のDiscordサーバーでの告知
- 既存の作家コミュニティ（エブリスタ作家の会等）への参加・紹介

**リクルーティング計画:**

- 目標人数: 作家30名（α版5名→β版30名）、読者100名（β版から参加）
- 期間: 開発期間中の6週目までにα版テスター確保、8週目までにβ版テスター確保
- 方法:
  - Week 1-2: ターゲット作家リスト100名作成、DM送付開始
  - Week 3-4: 応募フォーム設置、スクリーニング（作品PV数、連載頻度、海外展開意欲で選定）
  - Week 5-6: α版テスター5名確定、β版候補30名にウェイティングリスト登録依頼
  - Week 7-8: 読者募集（Reddit r/LightNovels、Discord翻訳コミュニティへの投稿）
