## 10. アクセシビリティ (Accessibility)

### 色のコントラスト

- [ ] すべてのテキストがWCAG AA基準を満たしている (コントラスト比 4.5:1以上)
- [ ] エラー表示は色だけでなくアイコンも併用

### キーボード操作

- [ ] Tabキーで全ての操作可能要素にフォーカス移動できる
- [ ] Enterキーでボタンを押下できる
- [ ] Escキーでモーダルを閉じられる

### スクリーンリーダー

- [ ] すべての画像にalt属性
- [ ] aria-labelを適切に設定
- [ ] フォーカス順序が論理的

## 14. AIレビューチェックリスト

Design Handoff作成後、AIに以下を確認依頼することを推奨します：

### 品質チェック項目

- [ ] **網羅性**: 全ての画面状態（Default/Hover/Active/Disabled/Error/Loading/Empty）が定義されているか
- [ ] **アクセシビリティ**: WCAG 2.1 AA基準（色のコントラスト、キーボード操作、スクリーンリーダー）が考慮されているか
- [ ] **レスポンシブ**: Desktop/Tablet/Mobileの挙動が明確か
- [ ] **インタラクション**: アニメーションのパラメータ（duration、easing）が数値で定義されているか
- [ ] **エラーハンドリング**: エラーメッセージの文言とトーンが[Tone and Voice](../43.01_Tone_and_Voice.md)に準拠しているか
- [ ] **一貫性**: Design Systemとの整合性があるか
- [ ] **実装可能性**: エンジニアが迷わず実装できる詳細度か

