## 🚀 テンプレートの使い方

### 1. テンプレートをコピー

```bash
# 例: PRDテンプレートをコピーして、SSO機能のPRDを作成
cp docs/_templates/31.01_PRD_<Function_Name>.md docs/30_Definition_Requirements/31.01_PRD_SSO.md
```

### 2. プレースホルダーを置き換え

#### ファイル名のプレースホルダー

| プレースホルダー    | 説明                                 | 例                                 |
| ------------------- | ------------------------------------ | ---------------------------------- |
| `<Function_Name>`   | 機能名（英数字、アンダースコア使用） | `SSO_Login`, `Analytics_Dashboard` |
| `<Persona_Name>`    | ペルソナ名                           | `Enterprise_Admin`, `End_User`     |
| `<Competitor_Name>` | 競合企業名                           | `Slack`, `Notion`                  |
| `<Title>`           | タイトル（簡潔に）                   | `Database_Choice`, `Auth_Strategy` |
| `<Date>`            | 日付（YYYY-MM-DD形式）               | `2025-01-27`                       |

#### ドキュメント内のプレースホルダー

| プレースホルダー      | 説明                       | 例                                               |
| --------------------- | -------------------------- | ------------------------------------------------ |
| `[Name]`              | 担当者名                   | `山田太郎`, `佐藤花子`                           |
| `[Link]`, `(link)`    | 実際のリンク               | `../30_Definition_Requirements/31.01_PRD_SSO.md` |
| `MM/DD`, `YYYY/MM/DD` | 日付                       | `01/27`, `2025/01/27`                            |
| `[機能名]`, `[説明]`  | 具体的な内容               | `シングルサインオン`, `SAML 2.0対応`             |
| `<!-- コメント -->`   | 記入ガイド（記入後は削除） |                                                  |

### 3. 内容を記入

- HTMLコメント `<!-- -->` は記入ガイドです。記入後は削除してください
- チェックボックス `- [ ]` は完了後に `- [x]` にマーク
- 不要なセクションは削除せず、「該当なし」や「N/A」と記入

### 4. 関連ドキュメントにリンク

- 相互参照を設定（例: PRD → Tech Spec → Design Handoff）
- 相対パスを使用: `../カテゴリ名/ファイル名.md`

### 5. レビュー・承認

- 各テンプレートの「承認」セクションに従って承認プロセスを実施
- GitLabのMRでレビューを実施

## 💡 よくある質問

### Q: テンプレートに該当するものがない場合は？

**A**: 以下の手順で新規テンプレートを作成してください：

1. 既存の類似テンプレートをコピー
2. 必要なセクションを追加・削除
3. このREADME.mdに新規テンプレートを追加
4. Librarianまたはチームにレビュー依頼

### Q: すべてのセクションを埋める必要がありますか？

**A**: いいえ。該当しないセクションは以下のように記入してください：

- **該当なし**: 該当するコンテンツがない場合
- **N/A**: このプロジェクトでは使用しない場合
- **TBD**: 後日決定する場合（決定期限を記入）

### Q: テンプレートを更新する場合は？

**A**:

1. このディレクトリのテンプレートファイルを直接編集
2. このREADME.mdの変更履歴に記録
3. 関連チームに通知（Slack、メール等）
4. 既存ドキュメントへの適用は任意（重要な変更の場合のみ適用）

### Q: カテゴリディレクトリが存在しない場合は？

**A**:

必要に応じてカテゴリディレクトリを作成してください：

```bash
# 例: 30_Definition_Requirementsディレクトリを作成
mkdir -p docs/30_Definition_Requirements

# カテゴリのREADME.mdも作成（_meta/README.mdを参考に）
```
