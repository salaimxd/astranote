# 60_Marketing_GTM - Index

## カテゴリ概要

マーケティング戦略、メッセージング、Go-To-Market計画、競合分析など、プロダクトを市場に届けるためのドキュメント群です。
営業・マーケティング活動の基盤となる資産を管理します。

---

## 対象職種

- **主要**: PMM, Sales
- **関係者**: PdM, CS

- **Messaging House**: 半年〜1年ごとに見直し

## 使い分けガイド

### Messaging House vs FAB Matrix

- **Messaging House**: プロダクト全体のメッセージング戦略（ブランドレベル）
- **FAB Matrix**: 個別機能の価値訴求（機能レベル）

### FAB Matrix vs PRD

- **PRD**: 技術的な機能仕様（What、内部向け）
- **FAB Matrix**: 顧客にとっての価値（Why、外部向け）

### Launch Plan vs PRD

- **PRD**: 機能の要件定義
- **Launch Plan**: 市場への届け方（GTM戦略、タスク管理）

### Battlecard vs Messaging House

- **Messaging House**: 自社の価値提案
- **Battlecard**: 競合との比較、勝ち筋

## 新規作成ガイド

### Messaging Houseを作成する場合

1. テンプレート: このカテゴリ内のファイルをテンプレートとして使用
2. 前提条件:
   - Vision/Missionが定義済み
   - Personaが明確
   - 競合分析が完了
3. 必須セクション:
   - ターゲットオーディエンス
   - アンブレラメッセージ（1文で表現）
   - 3本の柱（Core Pillars）
   - 価値提案
4. レビュープロセス:
   - PMM作成 → PdM/Sales Leadレビュー → 経営層承認

### FAB Matrixを更新する場合

1. 新機能リリース時
2. 既存機能の価値訴求を変更する時
3. 営業から「どう伝えればいいか分からない」という質問が来た時

### Launch Planを作成する場合

1. テンプレート: このカテゴリ内のファイルをテンプレートとして使用
2. 作成タイミング: ローンチ1ヶ月前
3. 必須セクション:
   - 目標・KPI
   - コミュニケーション戦略
   - チャネル戦略
   - タスクリスト（Marketing、Sales、CS、Product）
   - タイムライン
4. 関係者: PMM、PdM、Sales Lead、CS Lead、Engineer

### Battlecardを作成する場合

1. テンプレート: このカテゴリ内のファイルをテンプレートとして使用
2. 作成タイミング:
   - 主要競合が出現した時
   - 商談で競合に負けることが増えた時
3. 必須セクション:
   - 勝ち筋（当社の強み）
   - 競合の弱み（攻めるポイント）
   - キルポイント（質問集）
   - 反論処理
4. 更新頻度: 四半期ごと（競合情報は変化が速い）

## マーケティングメッセージングのベストプラクティス

### 良いメッセージングの特徴

- **顧客中心**: 自社の話ではなく、顧客の課題解決に焦点
- **具体的**: 曖昧な表現を避け、数字や具体例を使う
- **一貫性**: すべてのチャネルで同じメッセージ
- **差別化**: 競合とどう違うかが明確

### Feature vs Benefit の変換例

- ❌ **Feature**: "当社のSaaSにはSSO機能があります"
- ✅ **Benefit**: "1つのIDで全システムにログインでき、パスワード忘れの問い合わせがゼロになります"

## ローンチのベストプラクティス

### ローンチの成功要素

1. **明確な目標**: 何を達成したいか（リード獲得、アップセル、ブランド認知）
2. **ターゲットの明確化**: 既存顧客 vs 新規見込み客
3. **チャネルの選定**: メール、ブログ、SNS、PR、広告
4. **タイミング**: リリース日から逆算したタスク管理
5. **測定**: KPIを設定し、ローンチ後1週間、1ヶ月で振り返り

### よくある失敗パターン

- ❌ **機能リリースだけで終わる** → マーケ施策がない
- ❌ **内部連携不足** → Salesが知らない、CSが答えられない
- ❌ **KPI未設定** → 成功したか分からない

## よくある質問

### Q: Messaging Houseはいつ作成しますか？

**A**: プロダクト立ち上げ時、または大きなリポジショニング時。一度作成したら半年〜1年は基本的に変えません。

### Q: 競合が多い場合、Battlecardは全社分作りますか？

**A**: 主要競合3〜5社に絞ってください。それ以外は簡易版（1ページサマリ）で十分です。

### Q: Launch Planは誰が作成しますか？

**A**: PMMが主導しますが、PdM、Sales Lead、CS Leadと協力して作成します。全職種が関わるドキュメントです。

### Q: FAB Matrixはいつ使いますか？

**A**:

- 営業がピッチ資料を作る時
- マーケがLP（ランディングページ）を作る時
- CSがユーザーに価値を説明する時

## 営業・マーケティングの連携

### Salesとの連携ポイント

- **Battlecard**: 商談前に必ず確認
- **FAB Matrix**: ピッチトークに活用
- **Launch Plan**: 新機能リリース時の営業トレーニング

### CSとの連携ポイント

- **Messaging House**: 顧客説明時の価値訴求
- **FAB Matrix**: オンボーディング時の価値伝達
- **Launch Plan**: ヘルプセンター記事の作成
