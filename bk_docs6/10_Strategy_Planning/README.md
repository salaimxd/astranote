# 10_Strategy_Planning - Index

## カテゴリ概要

プロダクトの長期的な方向性と戦略を定義するドキュメント群です。
全チームメンバーが判断の指針とする最上位の文書を管理します。

- **主要**: PdM, EM, PMM

## 新規作成ガイド

### Vision/Missionを作成する場合

1. テンプレート: このカテゴリ内のファイルをテンプレートとして使用
2. ステークホルダー: 経営層、PdM、EM、PMM
3. レビュープロセス: 全社レビュー → 経営承認

### Roadmapを作成する場合

1. テンプレート: このカテゴリ内のファイルをテンプレートとして使用
2. 前提: Vision/Missionが定義済み
3. レビュープロセス: PdM/EM/PMM合意 → 全体共有

### OKRsを作成する場合

1. テンプレート: このカテゴリ内のファイルをテンプレートとして使用
2. 前提: Roadmapが定義済み
3. レビュープロセス: チーム内合意 → 隔週進捗確認

## よくある質問

### Q: Vision/Missionはどのくらいの頻度で見直すべきですか？

**A**: 基本的には半年〜1年ごと。ただし、大きな市場変化や戦略転換があった場合は随時見直してください。

### Q: Roadmapに記載する粒度は？

**A**: 主要機能レベル。詳細な仕様はPRDに記載します。四半期単位で「何をリリースするか」が分かる粒度が目安です。

### Q: OKRsのObjectiveはいくつ設定すべきですか？

**A**: 3〜5個を推奨。多すぎると焦点がぼやけます。各Objectiveに対して2〜3個のKey Resultsを設定してください。
