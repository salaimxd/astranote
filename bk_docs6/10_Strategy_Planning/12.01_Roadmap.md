# ロードマップ

> [!NOTE]
> プロダクトフェーズに沿った主要機能のリリース計画を可視化します。
> フェーズごとにレビューし、市場や組織の変化に応じて調整してください。
> **利用職種**: PdM, EM, PMM, Design Lead

---

## 全体テーマ

> **Foundation & Growth**
> 技術基盤の構築から、初期ユーザーの獲得、そしてグローバル展開への足がかりを作る。

### 背景

Astranoteは新規立ち上げフェーズにあるため、まずはPMF（Product-Market Fit）の検証が最優先事項となる。
段階的に機能をリリースし、フィードバックループを回しながら成長を目指す。

## フェーズ別マイルストーン

| Phase | 名称              | 目標                             | マネタイズ         |
| ----- | ----------------- | -------------------------------- | ------------------ |
| **0** | **Foundation**    | 基盤構築・設計                   | なし               |
| **1** | **Closed Beta**   | 初期フィードバック収集           | なし               |
| **2** | **Open Beta**     | 初期ユーザー獲得・PMF検証        | 広告のみ           |
| **3** | **Growth**        | ユーザー基盤拡大・収益化開始     | 広告 + サブスク    |
| **4** | **Scale**         | グローバル展開・本格収益化       | フルマネタイズ     |

## 戦略的優先事項

1. **翻訳品質の確立**: ラノベ特有の表現に対応できるAI翻訳基盤の構築
2. **作家体験の最適化**: 執筆から翻訳、公開までのシームレスなフロー
3. **グローバルコミュニティ**: 言語を超えた交流の促進

## 主要機能のロードマップ (詳細)

### Phase 0: Foundation (基盤構築)

**ステータス**: 完了/進行中
**目標**: 技術基盤の構築、設計ドキュメント整備

| 機能名             | 説明                     | ステータス |
| ------------------ | ------------------------ | ---------- |
| 技術スタック決定   | Next.js, Supabase等      | Done       |
| データベース設計   | Schema策定               | Done       |
| API設計            | REST/GraphQL             | Done       |
| MVP機能定義        | 要件定義                 | Done       |

### Phase 1: Closed Beta (期間: 2-4週間)

**ステータス**: 未着手
**目標**: 招待作家からのフィードバック収集、プロダクト改善

| 機能名     | 説明                         | 担当 |
| ---------- | ---------------------------- | ---- |
| 作家機能   | アカウント作成、作品投稿、エディタ | Dev  |
| 翻訳機能   | 日本語→英語の自動翻訳       | AI   |
| 読者機能   | 閲覧、ブックマーク           | Dev  |
| 分析       | 基本的なアクセス統計         | Dev  |

**KPI**:

- β参加作家数: 50-100人
- フィードバック回収率: 80%

### Phase 2: Open Beta / 正式ローンチ

**ステータス**: 未着手
**目標**: 初期ユーザー獲得、PMF検証

| 機能名     | 説明                     | 担当 |
| ---------- | ------------------------ | ---- |
| 作家機能   | 用語集管理、基本統計     | Dev  |
| 翻訳機能   | 日本語→英語（広告付き） | AI   |
| 読者機能   | コメント、フォロー       | Dev  |
| 収益化     | 広告表示                 | Biz  |

**KPI**:

- 登録作家数: 500人
- MAU（読者）: 5,000人
- 翻訳利用率: 80%

### Phase 3: Growth (成長期)

**ステータス**: 未着手
**目標**: ユーザー基盤拡大、マネタイズ開始

| 機能名       | 説明                             | 担当 |
| ------------ | -------------------------------- | ---- |
| 追加言語     | 韓国語、中国語（繁体）対応       | AI   |
| 読者サブスク | 広告非表示、オフライン閲覧       | Dev  |
| 投げ銭       | 作家への直接支援                 | Dev  |
| 先読み       | 作家が設定可能な有料先読み       | Dev  |

**KPI**:

- 登録作家数: 5,000人
- MAU（読者）: 50,000人
- 月間投げ銭総額: 100万円

### Phase 4: Scale (スケール期)

**ステータス**: 未着手
**目標**: グローバル展開、本格収益化

- **全言語**: 主要言語すべて対応
- **作家サブスク**: Pro/Businessプラン
- **API**: 外部サービス連携

**KPI**:

- MAU: 500,000人
- MRR: 1,000万円以上
- グローバル比率: 60%以上

## リソース配分計画

- **エンジニアリング**: 60% 新規機能開発（翻訳エンジン含む）、30% フロントエンド/UX、10% インフラ
- **デザイン**: 50% UI/UX、30% プロモーション素材、20% ユーザーリサーチ

## リスクと依存関係

| リスク                   | 影響度 | 対応策                                       |
| ------------------------ | ------ | -------------------------------------------- |
| 翻訳APIのコスト増大      | High   | キャッシング、自社モデルの検討               |
| 翻訳品質の低評価         | High   | 用語集機能の強化、フィードバックループ       |
| 競合サービスの台頭       | Medium | 独自コミュニティの形成、先行者利益           |
