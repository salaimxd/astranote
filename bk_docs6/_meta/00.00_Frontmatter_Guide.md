# フロントマター記入ガイド

> [!NOTE]
> 全ドキュメントのフロントマター（YAMLヘッダー）に記入する各フィールドの定義と使い方を説明します。
> 新規ドキュメント作成時やメタデータ更新時に参照してください。

---

## フロントマターとは

ドキュメントの冒頭に記述するYAML形式のメタデータです。ドキュメントの分類、検索、管理に使用します。

**基本構造:**

```yaml
---
title: ドキュメントのタイトル
type: ドキュメントタイプ
status: ステータス
role: 主な利用職種
tags: [タグ1, タグ2, タグ3]
---
```

## 必須フィールド

### title（タイトル）

**定義**: ドキュメントの正式名称

**記入ルール**:

- 日本語または英語で記述
- 簡潔かつ内容を端的に表現
- プレースホルダー（`<Function_Name>`等）を含む場合あり（テンプレートファイル）

**例**:

```yaml
title: 'PRD: SSO機能'
title: チームの役割分担と承認フロー定義
title: 技術仕様書：WebSocket通信
```

### type（ドキュメントタイプ）

**定義**: ドキュメントの種別を示す分類

**使用可能な値**:

| 値             | 説明                 | 使用場所                      |
| -------------- | -------------------- | ----------------------------- |
| `meta`         | メタ情報・運用ルール | `_meta/`                      |
| `template`     | テンプレートファイル | `_templates/`                 |
| `strategy`     | 戦略・ビジョン       | `10_Strategy_Planning/`       |
| `research`     | 調査・分析           | `20_Discovery_Research/`      |
| `requirements` | 要件定義             | `30_Definition_Requirements/` |
| `design`       | デザイン仕様         | `40_Design_UX/`               |
| `technical`    | 技術仕様             | `50_Engineering_Tech/`        |
| `marketing`    | マーケティング       | `60_Marketing_GTM/`           |
| `enablement`   | 営業・サポート資料   | `70_Customer_Enablement/`     |

**例**:

```yaml
type: meta
type: template
type: requirements
```

### status（ステータス）

**定義**: ドキュメントの現在の状態

**メタファイル（\_meta）での使用可能な値**:

| 値           | 説明             | 使用タイミング             |
| ------------ | ---------------- | -------------------------- |
| `active`     | 現在使用中・有効 | 運用中のルールやガイド     |
| `deprecated` | 非推奨・廃止予定 | 古いルールで今後使用しない |
| `archived`   | アーカイブ済み   | 参照のみ、更新なし         |

**通常ドキュメントでの使用可能な値**:

| 値           | 説明           | 使用タイミング |
| ------------ | -------------- | -------------- |
| `draft`      | 下書き・作成中 | 初稿作成中     |
| `review`     | レビュー中     | レビュー依頼後 |
| `approved`   | 承認済み       | 承認者の承認後 |
| `published`  | 公開済み       | マージ・公開後 |
| `deprecated` | 非推奨         | 古いバージョン |
| `archived`   | アーカイブ     | 参照のみ       |

**テンプレートファイル（\_templates）での値**:

| 値       | 説明                     |
| -------- | ------------------------ |
| `active` | 現在使用中のテンプレート |

**例**:

```yaml
status: active
status: draft
status: review
status: published
```

### role（主な利用職種）

**定義**: このドキュメントを主に使用する職種や権限レベル

**使用可能な値**:

| 値         | 説明                                 |
| ---------- | ------------------------------------ |
| `Admin`    | 管理者・全員                         |
| `PdM`      | プロダクトマネージャー               |
| `EM`       | エンジニアリングマネージャー         |
| `Engineer` | エンジニア                           |
| `TechLead` | 技術リード                           |
| `Designer` | デザイナー                           |
| `PMM`      | プロダクトマーケティングマネージャー |
| `Sales`    | 営業                                 |
| `CS`       | カスタマーサクセス                   |
| `QA`       | 品質保証                             |
| `SRE`      | サイト信頼性エンジニア               |

**記入ルール**:

- 主な利用職種を記載（複数ある場合はカンマ区切り、または最も重要な職種のみ）
- `Admin`は全職種が対象の場合に使用

**例**:

```yaml
role: Librarian
role: PdM
role: Engineer, TechLead
```

### tags（タグ）

**定義**: ドキュメントをカテゴリ別に分類するためのタグ

**記入ルール**:

- YAML配列形式で記述: `[タグ1, タグ2, タグ3]`
- 2〜3個程度が推奨
- 小文字・ハイフン区切り（例: `tech-spec`, `post-mortem`）
- 日本語タグも可（検索性を考慮）

### メタファイルの例

```yaml

title: チームの役割分担と承認フロー定義
type: meta
status: active
role: Librarian
tags: [team, workflow]

```

### テンプレートファイルの例

```yaml

```

## 更新ルール

### フロントマターを更新するタイミング

1. **status変更**
   - レビュー依頼時: `draft` → `review`
   - 承認時: `review` → `approved`
   - 公開時: `approved` → `published`
   - 廃止時: `published` → `deprecated` または `archived`

2. **tags追加**
   - 新しいカテゴリに該当するようになった場合
   - 検索性を向上させたい場合

3. **role変更**
   - 主な利用職種が変わった場合

### 更新時の注意事項

- フロントマターはYAML形式なので、**インデントや記号に注意**
- `tags`は必ず配列形式: `[tag1, tag2]`
- 日付フィールドは追加しない（Gitの履歴で管理）

## トラブルシューティング

### Q: タグは何個まで付けられますか？

**A**: 制限はありませんが、2〜3個が推奨です。多すぎると管理が煩雑になります。

### Q: 新しいtypeやstatusを追加したい場合は？

**A**: このドキュメントを更新し、チームで合意を取ってから使用してください。統一性が重要です。

### Q: roleに複数の職種を記載する方法は？

**A**: カンマ区切りで記載できますが、主要な職種1つのみの記載を推奨します。

```yaml
role: Engineer, TechLead
```
