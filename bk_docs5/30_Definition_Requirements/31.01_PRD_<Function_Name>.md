## 17. AIレビューチェックリスト

PRD作成後、AIに以下を確認依頼することを推奨します：

### 品質チェック項目

- [ ] **明確性**: 目的とスコープが明確に定義されているか
- [ ] **測定可能性**: KPIが定量的で測定可能か
- [ ] **優先順位**: Must/Should/Could/Won't Haveが明確に分類されているか
- [ ] **完全性**: エッジケース（エラー、空状態、権限不足）が考慮されているか
- [ ] **ユーザー中心**: ユーザーストーリーが具体的か
- [ ] **整合性**: Persona、Roadmap、User Interviewsとの矛盾がないか
- [ ] **セキュリティ**: セキュリティ/プライバシーリスクが洗い出されているか
- [ ] **実装可能性**: エンジニアが実装を開始できるレベルの詳細度か

### AIプロンプト例

```
以下のPRDを品質レビューしてください：

[PRDの内容を貼り付け]

評価基準（各項目を採点）：
1. 明確性: 目的とスコープが明確か
2. 測定可能性: KPIが定量的か
3. 完全性: Must/Should/Could/エッジケースが網羅されているか
4. 整合性: 他ドキュメントとの矛盾がないか
5. 実装可能性: エンジニアが実装を開始できるレベルか

各項目を採点し、具体的な改善提案を出してください。

参考: /home/salaim/work/product-docs/00_Meta/00.03_AI_Prompts.md
```
