## 🎨 ドキュメント作成のルール

### 命名規則
```
[カテゴリ番号].[サブカテゴリ番号]_[ドキュメント名].md

例:
- 31.01_PRD_SSO_Login.md
- 52.01_ADR_Database_Choice.md
```

### テンプレート使用
各カテゴリ内のファイルをテンプレートとして使用します。
プレースホルダー（`<Function_Name>`、`[Name]`、`(link)`等）を実際の内容に置き換えてください。

詳細: [Index.md#ドキュメント管理のルール](Index.md#ドキュメント管理のルール)

### プレースホルダー
- `<Function_Name>`: 機能名（例: SSO_Login）
- `[Name]`: 担当者名（例: 山田太郎）
- `(link)`: 実際のリンク（例: ../30_Definition_Requirements/31.01_PRD_SSO.md）
- `MM/DD`、`YYYY/MM/DD`: 日付（例: 2025/01/19）

## 📚 重要な用語（Glossary抜粋）

### ビジネス用語
- **PRD**: Product Requirements Document（要件定義書）
- **OKR**: Objectives and Key Results（目標管理フレームワーク）
- **PMF**: Product-Market Fit（プロダクトと市場の適合）
- **Churn Rate**: 解約率

### 技術用語
- **ADR**: Architecture Decision Record（アーキテクチャ決定記録）
- **SSO**: Single Sign-On（シングルサインオン）
- **API**: Application Programming Interface
- **CI/CD**: Continuous Integration / Continuous Delivery

### 役職
- **PdM**: Product Manager
- **EM**: Engineering Manager
- **PMM**: Product Marketing Manager
- **CS**: Customer Success
