## 1. 考えていること・感じていること（Think & Feel）

**考えていること:**

- 「海外の読者が『英語版を出してくれ』ってコメントしてくれるんだけど、どうやって届ければいいのか分からない」
- 「プロの翻訳者に頼んだら1話数万円。100話の長編だと数百万円。現実的じゃない」
- 「AI翻訳で自作を翻訳してみたけど、魔法名や固有名詞がめちゃくちゃ。これを自分の名前で公開するのは嫌だ」
- 「カクヨムの月3〜5万円のリワードでは副業にもならない。でも海外読者がつけば収益は何倍にもなるかも」
- 「Notionでプロット、VS Codeで原稿、カクヨムに投稿。毎回この行き来がだるい。全部1つで完結したい」
- 「執筆中にキャラ設定を確認するたびウィンドウを切り替える。100人超のキャラがNotionのページに散らばっていて探すのが大変」

**感じていること:**

- 期待: AI翻訳技術の進歩で、いつか自動で高品質に多言語化できる日が来るかも
- 喜び: 海外読者から「英語版ないの？」と聞かれる瞬間。自分の物語が言語を超えて価値を持つ証拠
- 不安: WebNovelのような搾取的プラットフォームに取り込まれてしまうリスク
- 懸念: AI翻訳の品質が低いまま公開すると、自分の作品のブランドが傷つくかもしれない
- フラストレーション: 「書く」以外の作業（ツール切り替え、コピペ、フォーマット修正）に時間を取られている

**心の中の声:**

> 「自分の作品を世界中の人に読んでもらいたい。でも翻訳のコストも品質も、自分一人じゃどうにもならない。そもそも、書くこと自体にもっと集中できる環境が欲しい。」

## 3. 言っていること・していること（Say & Do）

**言っていること:**

- 口癖: 「書きたいものがあるから書く。でも読んでもらえたらもっと嬉しい」
- よく話題にすること: カクヨムのリワード制度、書籍化の条件、海外読者からの反応
- SNSでの発信: 執筆進捗の報告、新章公開の告知、Web小説市場への意見

**具体的な引用:**

> 「海外の読者が『英語版を出してくれ』って言ってくれるのに、何もできないのがもどかしい。」

> 「WebNovelは収益の80%取るって聞いた。それなら無料で公開してた方がマシ。」

**していること:**

- 日常の行動: 平日夜21-24時に毎日2,000〜3,000字を執筆。コメントへの返信率90%以上
- 習慣: 毎朝カクヨムのアクセス解析をチェック。週末はプロット構成と推敲
- ワークフロー（3ツール横断の非効率な現状）:
  1. Notionでプロットとキャラクター設定を管理（100人超のキャラ、魔法体系、世界地図）
  2. VS Codeで原稿を執筆（Markdown形式）— カクヨムのエディタが貧弱なため
  3. カクヨムに手動コピペで投稿 → フォーマット崩れの修正 → コメント確認 → 読者の反応を次章に反映
- 執筆中のキャラ設定確認: VS Codeからウィンドウを切り替え → Notionで検索 → VS Codeに戻る（1回あたり30秒〜1分のロス、1日に10回以上）

## 5. 悩み・ストレスになっていること（Pain）

**主な悩み:**

1. **翻訳コストが非現実的** - プロ翻訳者は1話数万円。100話超の長編は総額数百万円。月1万円の予算では何もできない
2. **カクヨムの収益が少なすぎる** - 150万PVでも月3〜5万円。海外読者を獲得できれば大幅に増やせる可能性があるのに手段がない
3. **AI翻訳の品質が自作品に見合わない** - ファンタジー固有の用語（魔法名、種族名、地名）がDeepLでは不自然に翻訳される。自分の名前で公開できるレベルではない
4. **執筆環境が分断されている** - Notion（プロット管理）→ VS Code（原稿執筆）→ カクヨム（投稿）の3ツール横断。「書く」以外の作業に時間を取られ、限られた執筆時間がさらに圧迫される

**ストレスの原因:**

- 海外読者の「英語版ないの？」に「ない」としか答えられないもどかしさ
- WebNovelの搾取的条件（収益80%取得、著作権帰属）を知った上での選択肢のなさ
- 本業のエンジニア業務と執筆の両立による慢性的な時間不足
- Notionのプロット → VS Codeの原稿 → カクヨムの投稿画面を行き来する非効率さ
- カクヨムのエディタが貧弱で、VS Codeで書いてからコピペする二度手間
- 長編の設定管理（キャラクター100人超、魔法体系、世界地図）がNotionのページに散らばり、執筆中に参照しにくい

**障害・フラストレーション:**

- 翻訳に割ける時間がゼロ（本業 + 執筆で手一杯）
- 英語力TOEIC 650程度では、翻訳品質を自分で検証できない
- 既存の海外プラットフォームは作家にとって不利な条件ばかり
- 投稿時のコピペでフォーマットが崩れることがある（ルビ、傍点等）
- 「執筆に集中したいのに、ツールの操作に時間を取られる」という本末転倒

**リスク・不安:**

- 海外プラットフォームに作品を出して、著作権を失うリスク
- AI翻訳の品質が低い状態で公開して、海外での評判を落とすリスク
- カクヨムの読者を失うリスク（新プラットフォームに移行する場合のスイッチングコスト）

**克服したい課題:**

- 「高品質な翻訳」と「公正な収益分配」の両方を実現するプラットフォームを見つけたい

## 6. あったら嬉しいこと（Gain）

**望んでいること:**

1. **プロット管理・キャラ設定・原稿執筆・投稿がすべて1つの環境で完結する統合執筆体験**
2. 執筆中にキャラクター設定や世界観設定をサイドパネルで即参照できる機能
3. ワンクリックで作品が高品質に多言語翻訳される体験（固有名詞辞書のカスタマイズ付き）
4. 公正な収益分配（最低でも50%以上が作家に。著作権は作家が保持）
5. 言語圏ごとの読者反応を分析できるダッシュボード

**成功の定義:**

- プロットを確認しながら原稿を書き、投稿ボタンを押すだけで国内外の読者に届く状態
- 海外読者からの収益が月10万円を超える状態
- 複数言語で同時連載でき、各言語圏から読者のフィードバックが届く状態

**理想的な状態:**

- **統合エディタでプロットを確認しながら原稿を書く → 書き終えたら投稿ボタンを押すだけ** → 自動で英語・中国語等に高品質翻訳される → 海外読者からのコメントが翻訳されて届く → 翻訳版の収益もリアルタイムで確認できる

**潜在的なニーズ:**

- **プロット管理・設定資料・原稿執筆・投稿が一気通貫で行える統合執筆環境**
- **執筆中にキャラクター設定や世界観設定をサイドパネルで即参照できる機能**
- 翻訳結果の固有名詞辞書（魔法名、地名、種族名等）を作家自身が管理・カスタマイズできる機能
- 翻訳版のプレビュー機能（公開前に翻訳品質を確認・修正できる）
- 言語圏別の読者分析（どの国でどの章が人気か、離脱ポイントはどこか）

**期待する体験:**

- 「全部1つの環境で完結する」快適さ。プロットを横に出しながら原稿を書き、投稿ボタンを押すだけで世界に届く。執筆以外の作業に時間を取られない体験

## 得られたインサイト

**主要なインサイト:**

1. **作家の最大の障壁は「翻訳コスト」ではなく「翻訳品質のコントロール」** - 佐藤は翻訳そのものよりも、翻訳結果が自分の作品にふさわしいかどうかを気にしている。固有名詞辞書のカスタマイズが鍵
2. **「公正な収益分配」は作家の信頼獲得の最低条件** - WebNovelの80%搾取が業界の悪評を作っている。50%以上の作家取り分は必須の参入条件
3. **作家は「読者の反応」が最大のモチベーション** - 収益以上に、海外読者からの反応・コメントが作家の参加動機になる
4. **「執筆環境の分断」は日常的なフラストレーションの源泉** - 3ツール横断のワークフロー（Notion → VS Code → カクヨム）が、限られた執筆時間をさらに圧迫。統合執筆環境は翻訳とは独立した強い価値

**発見した潜在ニーズ:**

- 統合執筆環境: プロット管理・設定資料・原稿執筆・投稿を1つのツールで完結させたいニーズ
- サイドパネル参照: 執筆中にキャラ設定や世界観設定を即座に参照したいニーズ（ウィンドウ切り替え不要）
- 翻訳品質のカスタマイズ: 作家が固有名詞辞書を管理することで、AI翻訳の品質をコントロールしたいニーズ
- 多言語読者分析: 言語圏ごとの読者行動を可視化し、次章の執筆に活かしたいニーズ
- 段階的な海外展開: まず1作品だけ英語で試してみて、反応を見てから他言語に展開したいニーズ

**仮説:**

- 「プロット・設定・原稿・投稿が一気通貫で行える統合執筆環境は、3ツール横断の非効率を解消し、作家のプラットフォーム定着率を大幅に高める」
- 「固有名詞辞書のカスタマイズ機能があれば、AI翻訳品質への不安が大幅に軽減される」
- 「作家の収益分配率を60-70%に設定すれば、WebNovelからの移行動機になる」
- 「海外読者のコメント翻訳機能は、作家のエンゲージメントを大幅に高める」
