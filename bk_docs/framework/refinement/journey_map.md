# 📖 使い方ガイド

## ジャーニーマップとは

ジャーニーマップは、**ユーザーがプロダクトやサービスと関わる体験を時系列で可視化したツール**です。ユーザーの行動、感情、タッチポイント、ペインポイントを一つの図にまとめることで、体験全体を俯瞰し、改善の機会を発見できます。

## なぜ作るのか

**1. ユーザー視点での体験理解**
- プロダクト中心ではなく、ユーザー中心の視点で考える
- ユーザーが実際にどう感じているかを可視化

**2. ペインポイントの特定**
- ユーザーが困っている箇所を明確にする
- 体験のボトルネックを発見

**3. チーム全体での共通認識**
- 開発者、デザイナー、PM、マーケティングが同じ理解を持つ
- 議論の土台となる

**4. 改善の優先順位付け**
- どこから改善すべきかを判断する根拠
- インパクトの大きい箇所に集中

## ジャーニーマップの構成要素

### 1. ペルソナ
- 誰のジャーニーを描くか
- [Discovery のペルソナ](../discovery/persona.md)を活用

### 2. フェーズ（ステージ）
ユーザーが辿る主要な段階。例:
- **認知（Awareness）**: プロダクトを知る
- **検討（Consideration）**: 使うか判断する
- **購入・登録（Acquisition）**: 利用を開始する
- **利用（Usage）**: 実際に使う
- **継続・推奨（Retention/Advocacy）**: リピート利用や他者への推奨

### 3. タッチポイント
ユーザーがプロダクトと接する場所やチャネル
- ウェブサイト、アプリ、メール、カスタマーサポート、SNS など

### 4. ユーザーの行動
各フェーズで実際に何をするか
- 情報を検索する、登録する、支払う、問い合わせる など

### 5. ユーザーの感情
各ステップでどう感じているか（感情曲線）
- ポジティブ（期待、喜び、満足）
- ニュートラル
- ネガティブ（不安、困惑、不満）

### 6. ペインポイント
ユーザーが困っていること、障害になっていること
- 情報が見つからない、手続きが複雑、レスポンスが遅い など

### 7. 改善機会
どうすれば体験を向上できるか
- UI改善、情報追加、プロセス簡素化、パーソナライゼーション など


## 次のステップ

ジャーニーマップを作成したら、以下のステップでプロダクト改善につなげます。

### 1. [ユーザーストーリー](./user_stories.md)に変換

ペインポイントと改善機会を、実装可能なユーザーストーリー形式で記述します。

詳しい変換方法は、[ユーザーストーリー > ジャーニーマップから次のステップへ](./user_stories.md#ジャーニーマップから次のステップへ)を参照してください。

### 2. [テストケース（Gherkin）](./test_cases.md)に変換

ユーザーストーリーを実行可能なテストケースに変換し、品質を保証します。

### 3. 改善施策の実行と計測

- 優先度の高い改善から着手
- A/Bテストで効果を検証
- KPIで成果を測定
- ジャーニーマップを継続的に更新
