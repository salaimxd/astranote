---
title: ノーススターメトリック（NSM）
---

> このドキュメントは、プロダクトの本質的な価値を1つの指標で表現し、チーム全体の方向性を示すNorth Star Metricを設定するためのガイドです。

## NSMの特徴

優れたNSMは以下の特徴を持ちます。

### 1. 顧客価値を表現している

- 単なる売上やユーザー数ではない
- 顧客が「成功している」状態を示す
- プロダクトの価値提案と直結

### 2. ビジネス成長と相関している

- 収益と強い相関がある
- 持続可能な成長を示す
- 短期的な数字だけでなく、長期的な成功を予測

### 3. 行動可能である

- 施策の効果が測定できる
- A/Bテストで改善できる
- 各チームが貢献できる

### 4. シンプルで理解しやすい

- 1つの数字（複合指標ではない）
- エンジニア、デザイナー、ビジネス側が共通理解
- 新メンバーにも説明しやすい

## NSMの見つけ方

### Step 1: プロダクトの価値提案を明確にする

- プロダクトのコアバリューは何か？
- 顧客はどんな「成功」を求めているか？
- 「Aha!モーメント」はどこか？

### Step 2: 候補指標をリストアップ

| プロダクト | 候補指標 |
|---|---|
| **SaaS** | アクティブユーザー数、作成されたプロジェクト数、タスク完了数 |
| **Eコマース** | 購入数、購入金額、リピート購入率 |
| **メディア** | 記事閲覧数、滞在時間、シェア数 |
| **マーケットプレイス** | 取引総額（GMV）、取引数、アクティブバイヤー数 |

### Step 3: 評価基準で絞り込む

- [ ] 顧客価値を表現しているか？
- [ ] ビジネス成長と相関しているか？
- [ ] チームが影響を与えられるか？
- [ ] シンプルで理解しやすいか？
- [ ] 計測可能か？

### Step 4: データで検証

- 過去データで相関を確認
- 顧客インタビューで価値を確認
- チームで議論し、合意形成

## プロダクトタイプ別のNSM

### SaaS（プロジェクト管理ツール）

**North Star Metric: 週次アクティブチーム数（Weekly Active Teams）**

- なぜ: チームが活発に使っている = 価値を得ている
- ビジネス成長: アクティブチーム数 → 有料プラン転換 → 収益
- 行動可能: オンボーディング改善、機能追加、通知最適化

### Eコマース

**North Star Metric: 月次リピート購入者数（Monthly Repeat Purchasers）**

- なぜ: リピート購入 = 顧客満足度が高い
- ビジネス成長: リピート購入者 → LTV向上 → 収益増
- 行動可能: 商品品質、配送体験、リテンションメール

### メディア・コンテンツ

**North Star Metric: 週次エンゲージメント時間（Weekly Engaged Time）**

- なぜ: コンテンツに価値を感じている時間
- ビジネス成長: エンゲージメント → 広告収益、サブスク転換
- 行動可能: コンテンツ品質、レコメンデーション、UI/UX

### マーケットプレイス

**North Star Metric: 月次取引総額（Monthly GMV - Gross Merchandise Value）**

- なぜ: 取引が成立 = 買い手と売り手の両方が価値を得ている
- ビジネス成長: GMV → 手数料収益
- 行動可能: マッチング品質、検索、決済体験

### コミュニティ・ソーシャル

**North Star Metric: 週次アクティブユーザー（Weekly Active Users）**

- なぜ: 活発にコミュニケーション = ネットワーク価値
- ビジネス成長: WAU → ネットワーク効果 → 広告収益
- 行動可能: コンテンツ品質、通知、機能追加

## NSMの設定プロセス

### Step 1: チームでワークショップ

**参加者:**
- プロダクトマネージャー
- エンジニア
- デザイナー
- マーケティング
- 経営陣

**アジェンダ:**
1. プロダクトの価値提案を再確認
2. NSM候補をブレインストーミング
3. 評価基準で絞り込み
4. Input Metrics（KPI）をリストアップ

### Step 2: データで検証

- 候補指標と収益の相関を確認
- コホート分析でリテンションを確認
- ファネル分析でボトルネックを特定
- 顧客インタビューで価値を確認

### Step 3: 仮説として設定

- まずは仮説として設定
- 3ヶ月運用して評価
- 必要に応じて変更

### Step 4: ダッシュボード構築

- NSMをリアルタイムで追跡
- Input Metricsを分解
- チーム全体で共有

### Step 5: 定期的にレビュー

**週次:**
- NSMの推移を確認
- 異常値の分析
- 施策の効果測定

**月次:**
- トレンド分析
- セグメント別分析
- 次月の目標設定

**四半期:**
- NSMの妥当性を見直し
- 必要に応じて変更
