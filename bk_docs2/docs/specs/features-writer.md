## 概要

作家がStar Novelで小説を投稿し、グローバルに発信するための機能群です。

:::note[対象ユーザー]
[ユーザー定義](/product/users/) の「日本人Web小説作家」を参照
:::

---

## アカウント管理

### 機能概要

| 項目 | 内容 |
|------|------|
| 機能ID | WRITER_ACCOUNT |
| Phase | 1 |
| 優先度 | Must |

### 機能一覧

#### 新規登録

| 項目 | 仕様 |
|------|------|
| メール登録 | メールアドレス + パスワード |
| OAuth | Google、Twitter/X、GitHubに対応 |
| 必須項目 | メールアドレス、パスワード（8文字以上） |
| 任意項目 | ペンネーム、プロフィール画像 |
| 確認フロー | メール認証（24時間有効） |

#### ログイン

| 項目 | 仕様 |
|------|------|
| 認証方式 | メール/パスワード または OAuth |
| セッション | JWT（7日間有効、リフレッシュトークン30日） |
| 二要素認証 | Phase 2 で対応予定 |

#### プロフィール設定

| 項目 | 仕様 |
|------|------|
| ペンネーム | 2-30文字、日本語・英語・数字対応 |
| プロフィール画像 | 最大5MB、JPEG/PNG/GIF |
| 自己紹介 | 最大1,000文字 |
| SNSリンク | Twitter/X、Pixiv、なろう、カクヨム等 |
| 公開設定 | 公開/非公開の選択 |

### ユーザーストーリー

```
作家として、
メールアドレスで簡単に登録できる
ことで、すぐに執筆を開始できる。

作家として、
既存のなろう・カクヨムアカウントとリンクできる
ことで、読者に自分の実績を示せる。
```

## 作品管理

### 機能概要

| 項目 | 内容 |
|------|------|
| 機能ID | NOVEL_MANAGEMENT |
| Phase | 1 |
| 優先度 | Must |

### 作品作成

| 項目 | 仕様 |
|------|------|
| タイトル | 1-100文字 |
| サブタイトル | 0-200文字（任意） |
| あらすじ | 1-2,000文字 |
| カバー画像 | 推奨: 600x800px、最大10MB |
| ジャンル | 1つ必須、サブジャンル3つまで |
| タグ | 最大10個、自由入力 |
| 年齢制限 | 全年齢/R-15/R-18 |
| 連載状態 | 連載中/完結/休止 |

### ジャンル一覧

| メインジャンル | サブジャンル例 |
|---------------|---------------|
| ファンタジー | 異世界転生、異世界転移、ハイファンタジー、ローファンタジー |
| SF | 宇宙、近未来、ディストピア、サイバーパンク |
| ラブコメ | 学園、職場、幼馴染、ラブコメ |
| ホラー | ホラー、サスペンス、ミステリー |
| 歴史 | 戦国、幕末、中世ヨーロッパ |
| 現代 | 日常、青春、社会派 |
| その他 | エッセイ、詩、短編集 |

### 公開設定

| 設定 | 説明 |
|------|------|
| 下書き | 自分のみ閲覧可能 |
| 限定公開 | URLを知っている人のみ |
| 公開 | 全ユーザーに公開 |
| 予約公開 | 指定日時に自動公開 |

### ユーザーストーリー

```
作家として、
作品のジャンルとタグを細かく設定できる
ことで、読者に見つけてもらいやすくなる。

作家として、
予約公開機能を使える
ことで、定期更新のスケジュール管理ができる。
```

## 章管理

### 機能概要

| 項目 | 内容 |
|------|------|
| 機能ID | CHAPTER_MANAGEMENT |
| Phase | 1 |
| 優先度 | Must |

### 章の作成・編集

| 項目 | 仕様 |
|------|------|
| タイトル | 0-100文字（任意） |
| 本文 | 最大100,000文字/章 |
| 章番号 | 自動採番（手動変更可） |
| 並び替え | ドラッグ＆ドロップ |
| 前書き | 0-1,000文字（任意） |
| 後書き | 0-1,000文字（任意） |

### 公開オプション

| 項目 | 仕様 |
|------|------|
| 即時公開 | 保存と同時に公開 |
| 予約公開 | 日時指定（1分単位） |
| 有料設定 | Phase 3で対応（先読み課金等） |

### 自動保存

| 項目 | 仕様 |
|------|------|
| 保存間隔 | 30秒ごと |
| 保存先 | ローカルストレージ + サーバー |
| 履歴保持 | 直近10件の編集履歴 |
| 復元機能 | 任意の履歴に復元可能 |

## WYSIWYGエディタ

### 機能概要

| 項目 | 内容 |
|------|------|
| 機能ID | WYSIWYG_EDITOR |
| Phase | 1 |
| 優先度 | Must |

### 対応フォーマット

#### テキスト装飾

| 機能 | ショートカット | 説明 |
|------|---------------|------|
| 太字 | Ctrl/Cmd + B | **太字** |
| 斜体 | Ctrl/Cmd + I | *斜体* |
| 下線 | Ctrl/Cmd + U | <u>下線</u> |
| 取り消し線 | - | ~~取り消し線~~ |
| 傍点 | - | 強調用の点 |

#### ルビ機能（重要）

| 機能 | 記法 | 表示例 |
|------|------|--------|
| ルビ入力 | `|漢字《かんじ》` | 漢字（かんじ） |
| 一括ルビ | 選択→ルビボタン | GUI操作 |
| ルビ辞書 | 登録した読みを自動適用 | 一括変換 |

```
【ルビの重要性】
日本のライトノベルでは、ルビが表現の重要な要素。
例: |異世界《いせかい》、|魔法《まほう》、|勇者《ゆうしゃ》

翻訳時の処理:
- ルビは翻訳されない（元の読みを保持）
- または、読者の言語設定に応じて表示/非表示
```

#### 段落・構造

| 機能 | 説明 |
|------|------|
| 見出し | H2, H3 レベル |
| 段落 | 通常の文章ブロック |
| 区切り線 | シーン切り替え用 |
| 引用 | 引用ブロック |
| コードブロック | 固定幅フォント（魔法詠唱等） |

#### 特殊表現

| 機能 | 用途 |
|------|------|
| センタリング | タイトル、章名等 |
| 右寄せ | 手紙の署名等 |
| 字下げ | インデント調整 |
| 挿絵 | 画像挿入（Phase 2） |

### 文字数カウント

| 項目 | 表示 |
|------|------|
| 現在の章 | リアルタイム表示 |
| 作品全体 | ダッシュボードに表示 |
| 目標文字数 | 設定可能、進捗バー表示 |

### ユーザーストーリー

```
作家として、
ルビを簡単に入力できる
ことで、ラノベ特有の表現を維持できる。

作家として、
自動保存機能がある
ことで、執筆中のデータ消失を防げる。
```

## 用語集管理

### 機能概要

| 項目 | 内容 |
|------|------|
| 機能ID | GLOSSARY_MANAGEMENT |
| Phase | 2 |
| 優先度 | Should |

### 用語登録

| 項目 | 仕様 |
|------|------|
| 原語 | 日本語での表記 |
| 読み | ルビ/発音 |
| 英語訳 | 翻訳先の表記 |
| 説明 | 用語の解説（任意） |
| カテゴリ | 人名/地名/魔法/アイテム/その他 |

### 用語集の適用

```
┌─────────────────────────────────────────────────────┐
│           用語集適用フロー                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  【投稿時】                                         │
│  1. 作家が章を投稿                                  │
│  2. システムが用語を検出                            │
│  3. 登録済み用語 → 指定訳を適用                    │
│  4. 未登録用語 → AI翻訳を適用                      │
│                                                     │
│  【効果】                                           │
│  - 固有名詞の一貫性を保持                          │
│  - 「異世界」→「Another World」のような統一        │
│  - キャラ名のブレを防止                            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 用語集テンプレート

| テンプレート | 内容 |
|-------------|------|
| 異世界ファンタジー | 勇者、魔王、スキル、ステータス等 |
| 学園ラブコメ | 先輩、後輩、部活、文化祭等 |
| SF | 宇宙船、AI、コロニー等 |

### ユーザーストーリー

```
作家として、
キャラ名の英語表記を統一できる
ことで、読者が混乱しない翻訳を提供できる。

作家として、
用語集テンプレートを使える
ことで、一から登録する手間を省ける。
```

## アナリティクス

### 機能概要

| 項目 | 内容 |
|------|------|
| 機能ID | WRITER_ANALYTICS |
| Phase | 3 |
| 優先度 | Should |

### 閲覧統計

| 指標 | 説明 |
|------|------|
| PV（ページビュー） | 章ごと/作品全体の閲覧数 |
| UU（ユニークユーザー） | 訪問者数 |
| 読了率 | 章を最後まで読んだ割合 |
| 離脱ポイント | どこで読者が離脱したか |

### 読者属性

| 属性 | 説明 |
|------|------|
| 言語 | 閲覧言語の内訳 |
| 地域 | 国/地域の分布 |
| デバイス | PC/スマホ/タブレットの割合 |
| 流入元 | SNS/検索/ダイレクト等 |

### 時系列分析

| 分析 | 説明 |
|------|------|
| 日次推移 | 日ごとのPV/UU |
| 更新効果 | 新章公開後のアクセス増加 |
| 曜日傾向 | 読まれやすい曜日 |
| 時間帯傾向 | 読まれやすい時間帯 |

## 収益ダッシュボード

### 機能概要

| 項目 | 内容 |
|------|------|
| 機能ID | REVENUE_DASHBOARD |
| Phase | 3 |
| 優先度 | Should |

### 収益源

| 収益源 | 説明 | 作家取り分 |
|--------|------|-----------|
| 投げ銭 | 読者からの直接支援 | 80% |
| 広告収益分配 | PVに応じた分配 | 70% |
| サブスク収益分配 | 読者プレミアムからの分配 | 60% |

### 表示項目

| 項目 | 説明 |
|------|------|
| 今月の収益 | リアルタイム集計 |
| 累計収益 | 過去の総収益 |
| 振込履歴 | 過去の振込記録 |
| 振込予定 | 次回振込予定額 |

### 振込条件

| 項目 | 条件 |
|------|------|
| 最低振込額 | ¥1,000 |
| 振込サイクル | 月末締め、翌月15日払い |
| 振込手数料 | Star Novel負担 |
| 対応銀行 | 日本国内銀行、PayPal |

## 非機能要件

### パフォーマンス

| 項目 | 要件 |
|------|------|
| エディタ読み込み | 2秒以内 |
| 章保存 | 3秒以内 |
| 自動保存 | 1秒以内（バックグラウンド） |
| 画像アップロード | 5秒以内（5MBまで） |

### 可用性

| 項目 | 要件 |
|------|------|
| 稼働率 | 99.5%以上 |
| データバックアップ | 日次（30日保持） |
| 障害復旧 | 4時間以内 |

### セキュリティ

| 項目 | 要件 |
|------|------|
| 通信 | HTTPS必須 |
| 認証 | JWT + リフレッシュトークン |
| データ保護 | 作品データの暗号化 |
| 不正アクセス防止 | レート制限、CAPTCHA |
