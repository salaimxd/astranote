:::danger[このドキュメントについて]
このドキュメントは画面遷移と状態遷移を定義するためのテンプレートです。
ユーザーの操作フローと画面間の遷移を明確にしてください。
:::

画面遷移フロー、各画面の詳細仕様(目的、表示要素、遷移先、状態)、グローバル・ローカル状態管理、エラーハンドリングを定義します。ユーザー体験の一貫性を保ち、開発とデザインの認識合わせに活用します。

## 画面遷移フロー

### 初回起動フロー

```
スプラッシュ → オンボーディング → ログイン/新規登録 → ホーム
```

### メイン機能フロー

```
ホーム → 機能選択 → 機能詳細 → 結果表示 → ホーム
```

## 画面詳細仕様

### HOME: ホーム画面

**目的**: ユーザーの主要な操作の起点

**表示要素**:
- ヘッダー（ロゴ、通知アイコン、プロフィールアイコン）
- メインコンテンツエリア
- ナビゲーションバー（タブ形式）

**遷移先**:
- 機能A画面
- 機能B画面
- 設定画面
- プロフィール画面

**状態**:
- ローディング状態
- エラー状態
- 通常表示状態

## 状態管理

### グローバル状態

- ユーザー情報（ログイン状態、プロフィール）
- アプリ設定
- 通知状態

### ローカル状態

- 入力フォームの値
- UI表示状態（モーダル、ドロワー）
- ページネーション状態

## エラーハンドリング

| エラー種別 | 表示メッセージ | ユーザーアクション |
|-----------|---------------|-------------------|
| ネットワークエラー | 「通信エラーが発生しました」 | リトライボタン |
| 認証エラー | 「再ログインしてください」 | ログイン画面へ遷移 |
| データ取得失敗 | 「データの取得に失敗しました」 | リロードボタン |
